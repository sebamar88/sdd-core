# SSD-Core v0.1 Closure Week Plan

Date: 2026-05-03
Status: closed
Owner: core maintainers

## Goal

Close the v0.1 production-candidate stage with explicit evidence, stable scope, and documented acceptance criteria.

Official v0.1 scope includes concrete adapter manifests for:

- Codex
- Claude Code
- Gemini CLI
- OpenCode
- Qwen Code

## Working Principles

All tasks in this plan must follow:

- DRY
- KISS
- YAGNI
- SOLID
- GRASP
- LoD

## Exit Criteria

- CI passes on Windows, macOS, and Linux using `python scripts/release_check.py`.
- npm publish workflow succeeds with `NPM_REPOSITORY_TOKEN`.
- README, readiness docs, and templates state the same scope and non-goals.
- Validation and tests cover the key failure paths for `validate`, `check`, `sync-specs`, and `archive`.
- Remaining gaps are documented as post-v0.1 backlog items.

## Risks And Mitigations

- Risk: npm token exists but is inaccessible in workflow context.
  Mitigation: keep explicit token validation step and test via `workflow_dispatch`.

- Risk: validator work expands scope too much.
  Mitigation: enforce KISS and YAGNI, only high-signal checks for v0.1.

- Risk: docs drift again after fixes.
  Mitigation: update runtime docs and template docs in the same change set.
